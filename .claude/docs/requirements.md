# DevLaunch 要件定義書

## 1. 概要

- **アプリ名**: DevLaunch
- **対象プラットフォーム**: macOS 13 Ventura 以降
- **ターゲット構成**: 単一ターゲット（Developer ID 直接配布のみ）
- **Sandbox**: OFF（外部CLIコマンド実行のため）
- **配布方式**: OSS（MIT ライセンス）・GitHub Releases（DMG）
- **ウィンドウ管理**: メニューバー常駐アプリ（ポップオーバー表示）。リサイズなし。マルチウィンドウなし
- **ターゲットユーザー**: Claude Code / Codex / AI CLI を日常的に使う開発者（vibe coder 含む）
- **コンセプト**: プロジェクトフォルダをスキャンし、ワンクリックでエディタ＋AI CLIを起動するmacOSメニューバー常駐アプリ

### 1.1 プロジェクトの目的

- **完全無料・OSS（MIT ライセンス）** として公開する
- AI CLI を日常的に使う開発者の「エディタ＋AI CLI 起動」の手間を減らす
- 「非エンジニアが Claude Code だけで作った macOS アプリ」という制作プロセスそのものを公開し、同じように AI で開発したい人の参考にしてもらう

## 2. コア機能

### 2.1 フォルダスキャンによるプロジェクト自動検出

- **概要**: ユーザーが指定したフォルダ内のサブディレクトリをスキャンし、プロジェクトを自動検出する
- **検出条件**: `.git` ディレクトリが存在するサブディレクトリをプロジェクトと判定する
- **スキャン対象**: ユーザーが設定画面で指定した1つのフォルダ（v1では単一フォルダのみ）
- **初回起動時**: フォルダ選択ダイアログを即表示し、スキャン対象フォルダを設定させる
- **更新タイミング**: アプリ起動時にフルスキャン。以降は FSEvents でフォルダの変更を監視し、差分更新する。ポップオーバー表示時はキャッシュされた一覧を即表示する
- **ユーザー操作フロー**:
  1. メニューバーアイコンをクリック → ポップオーバーが開く
  2. プロジェクト一覧が表示される（フォルダ名をリスト表示）
  3. プロジェクト名をクリック → エディタ＋AI CLIが起動
- **制約**: 
  - スキャン深度は1階層のみ（指定フォルダ直下のサブディレクトリのみ）
  - 隠しフォルダ（`.`始まり）はスキャン対象外

### 2.2 ワンクリックでエディタ＋AI CLI同時起動

- **概要**: プロジェクト一覧からクリックするだけで、エディタと AI CLI を同時に起動する
- **エディタ起動**: 設定で指定されたエディタコマンドでプロジェクトフォルダを開く
  - デフォルト: `code`（VS Code）
  - 設定可能: `cursor`, `zed`, その他カスタムコマンド
- **AI CLI 起動**: 設定で指定された AI CLI を、エディタの統合ターミナルで実行する
  - デフォルト: `claude`（Claude Code）
  - 設定可能: `codex`, その他カスタムコマンド
  - 起動オプション: 自由入力（例: `--dangerously-skip-permissions`）
- **AI CLI 起動先**:
  - **エディタ統合ターミナル**（デフォルト）: エディタ内の統合ターミナルで AI CLI を実行
  - **外部ターミナル**（オプション）: macOS デフォルトのターミナルアプリで AI CLI を実行
  - 設定画面で切り替え可能
- **ユーザー操作フロー**:
  1. ポップオーバーのプロジェクト名をクリック
  2. エディタがプロジェクトフォルダを開く
  3. 同時に AI CLI が起動先（統合ターミナル or 外部ターミナル）で実行される
- **統合ターミナル起動の技術アプローチ**（優先度順）:
  1. **VS Code URI scheme**: `vscode://` カスタム URI でターミナルコマンドを送信。エディタ側の対応が必要
  2. **AppleScript キーストローク送信**: エディタにフォーカス → `Ctrl+`` でターミナルを開く → コマンド入力。アクセシビリティ権限が必要
  3. **シェルスクリプト経由**: `code [folder]` 実行後に `sleep` + `osascript` でキーストロークを送信
  - **フォールバック条件**: 上記のいずれも安定動作しない場合、外部ターミナルで起動する
  - **PoC 必須**: 実装計画の Phase 1 で上記アプローチの PoC を実施し、採用する方式を確定する
- **エラーハンドリング**:
  - エディタコマンドが見つからない場合: 「Editor not found. Please check Settings.」をポップオーバー内にインラインで表示
  - AI CLI コマンドが見つからない場合: 「AI CLI not found. Please check Settings.」をポップオーバー内にインラインで表示
  - スキャンフォルダが存在しない場合: 「Scan folder not found. Please update in Settings.」をポップオーバー内にインラインで表示し、フォルダ選択ダイアログへの導線を提供

### 2.3 エディタ・AI CLI・起動オプションの設定

- **概要**: ユーザーがエディタ、AI CLI、起動オプションを設定画面でカスタマイズできる
- **設定項目**:
  1. **スキャン対象フォルダ**: フォルダパスの指定（フォルダピッカーで選択）
  2. **エディタ**: プリセット選択（VS Code / Cursor / Zed）+ カスタムコマンド入力。デフォルト: VS Code（`code`）
  3. **AI CLI**: プリセット選択（Claude Code / Codex）+ カスタムコマンド入力。デフォルト: Claude Code（`claude`）
  4. **AI CLI 起動オプション**: 自由入力テキストフィールド（例: `--dangerously-skip-permissions`）
  5. **AI CLI 起動先**: 統合ターミナル / 外部ターミナル の選択（デフォルト: 統合ターミナル）
  6. **ソート順**: 最近使った順 / アルファベット順 の選択（デフォルト: 最近使った順）
  7. **ログイン時自動起動**: ON / OFF（デフォルト: OFF）
- **保存方式**: UserDefaults（@AppStorage でバインド）
- **ユーザー操作フロー**:
  1. メニューバーアイコンを右クリック or ポップオーバー内の歯車アイコン → 設定画面を開く
  2. 各項目を入力・選択
  3. 変更は即時反映（保存ボタン不要）

## 3. 課金設計

該当なし。完全無料・OSS。

## 4. AI機能

該当なし。このアプリは AI CLI を「起動する」ツールであり、AI 機能自体は持たない。

## 5. 機能要件詳細

### 5.1 UI/UX

- **ウィンドウ構成**: メニューバーポップオーバーのみ（通常ウィンドウなし）
- **ポップオーバーの内容**:
  - プロジェクト一覧（フォルダ名をリスト表示）
  - 各プロジェクト行: フォルダ名 + クリックで起動
  - 下部: 設定アイコン（歯車）
- **設定画面**: 別ウィンドウ（Settings ウィンドウ）で表示
- **テーマ**: システム追従（ダーク/ライトモードに自動対応）
- **Dock アイコン**: 非表示（メニューバーアプリのため `LSUIElement = true`）

### 5.2 キーボードショートカット

- **グローバルショートカット**: ポップオーバーの表示/非表示トグル
  - デフォルト: なし（ユーザーが設定画面で任意のキーを割り当て）
  - 実装: `CGEvent` / `NSEvent.addGlobalMonitorForEvents` で実装
- **アプリ内ショートカット**:
  - `⌘,`: 設定画面を開く（macOS 標準）
- **ポップオーバー内**:
  - `↑`/`↓`: プロジェクト選択
  - `Enter`: 選択中のプロジェクトを起動
  - `Esc`: ポップオーバーを閉じる

### 5.3 データ管理

- **保存方式**: UserDefaults
- **保存データ**:
  - スキャンフォルダパス
  - エディタコマンド
  - AI CLI コマンド・オプション
  - AI CLI 起動先設定
  - ソート順設定
  - ログイン時自動起動設定
  - グローバルショートカットのキー設定
  - 各プロジェクトの最終起動日時（DevLaunch 経由での起動時のみ記録。ソート用）
- **同期**: なし（ローカルのみ）
- **バックアップ**: なし（設定項目のみのため不要）

### 5.4 メニューバー

- **アイコン**: カスタムアイコン（SF Symbols ベース、`terminal` 系のシンボルを検討）
- **左クリック**: ポップオーバー表示
- **右クリック**: コンテキストメニュー（Settings... / Quit DevLaunch）

## 6. 非機能要件

- **対応言語**: 英語のみ
- **セキュリティ方針**: 
  - 個人情報を収集・保存しない
  - ネットワーク通信を行わない
- **権限要件**:
  - **アクセシビリティ権限**: 統合ターミナル起動方式が AppleScript キーストローク送信を採用する場合に必須。初回起動時にシステム設定への誘導ダイアログを表示する。外部ターミナルのみ使用する場合は不要
  - **フルディスクアクセス**: 不要（スキャン対象はユーザーが明示的に選択したフォルダのみ）
- **アクセシビリティ方針**: 
  - VoiceOver 基本対応（リスト項目のラベル設定）
  - キーボードナビゲーション対応
- **パフォーマンス目標**:
  - ポップオーバー表示: 0.3秒以内
  - プロジェクトスキャン: 1秒以内（100プロジェクト規模）
  - エディタ + AI CLI 起動: 2秒以内（プロセス起動まで）
- **オフライン対応**: 完全オフライン動作（ネットワーク不要）
- **公証（Notarization）**: 必須（Developer ID 配布のため）
- **Hardened Runtime**: 有効

## 7. リスクと対策

| リスク | 影響度 | 対策 |
|-------|--------|------|
| 統合ターミナル起動の安定性 | 中 | PoC で検証。不安定なら外部ターミナルにフォールバック |
| Raycast が AI CLI 統合拡張を出す | 低 | OSS 無料のため影響小。スタンドアロンアプリとしての価値を維持 |
| Claude Code / Codex 側が起動機能を内蔵 | 低 | 複数 AI CLI の統合ハブとしての価値が残る |
| 一人開発の持続性 | 低 | 機能がシンプルでメンテナンス負荷は最小限 |

## 8. v2 候補機能

- **Git 情報の表示**: ブランチ名、最終コミット日時をプロジェクト行に表示
- **プロジェクトごとのアイコン・カラー設定**: 視覚的な識別性向上
- **プロジェクトごとの AI CLI・オプション設定**: プロジェクトに応じた起動設定
- **複数スキャンフォルダ対応**: 複数の親フォルダからプロジェクトを検出
- **プロジェクトのメモ・タグ機能**: 整理・検索性の向上
- **ターミナルアプリの選択**: iTerm2, Warp, Ghostty 等への対応
- **検索・フィルタ機能**: プロジェクト名でインクリメンタル検索
- **Gemini CLI, Aider 等への対応**: 他の AI CLI ツールのサポート
